# Process Overview

## 1. Discovery

- Synthesize fragmented research evidence before locking a JTBD when raw inputs are spread across multiple sources.
- Define the problem, audience, primary JTBD, and constraints.
- Identify assumptions, risks, and unknowns.
- Decide whether to proceed, pivot, or stop.

## 2. Definition (Feature-Level PRD)

- Produce a PRD for feature-level work with business context, JTBD framing, MoSCoW scope, requirements, and success metrics.
- Keep unknowns explicit as open questions (`TBD`) instead of guessing.
- Align stakeholders on decisions, dependencies, and rollout expectations.

## 3. Ticket Breakdown and Acceptance Criteria

- Break approved PRD scope into tickets/stories that preserve the primary JTBD of each delivery slice.
- Attach ticket/story-level acceptance criteria to each implementation item.
- Ensure ticket criteria trace back to PRD requirements.

## 4. Delivery

- Execute implementation slices and initiatives.
- Validate ticket completion against acceptance criteria.
- Track feature outcomes against PRD success metrics.

## 5. Review

- Measure outcomes and guardrails.
- Capture learnings and meaningful decisions for future cycles.
