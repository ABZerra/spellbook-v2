# Glossary

## PRD
Product Requirements Document describing problem, users, scope, and success metrics.

## JTBD
Jobs To Be Done, a product framing method that captures the progress an actor is trying to make in a given situation.

## Acceptance Criteria
Testable statements defining when a requirement is considered complete from a user perspective.

## Discovery
Work done before delivery to validate needs, constraints, and solution direction.

## Initiative
Checkpoint tied to measurable outcomes in a plan.
