# Global Codex Defaults

- Prefer repo-local guidance over global guidance whenever they overlap.
- Keep instructions small and load detail on demand through repo docs and skills.
- Prefer modular boundaries when project complexity or reuse warrants them.
- Run the relevant verification before commit or PR whenever the repo defines it.
- Keep changes reviewable with clear summaries, risk notes, and focused diffs.
- Put hard rules in CI or validation scripts instead of relying on prose alone.
- Prefer the next relevant skill when a workflow naturally advances.
