# PAAL Core Outputs

`docs/outputs/` stores generated artifacts produced when PAAL is used on PAAL itself.

- Files here are self-hosted PAAL outputs, not skill fixtures.
- Per-skill fixtures belong under `skills/*/examples/`.
- Downstream project repositories use semantically named `docs/` subdirectories such as `docs/prd/` and `docs/context/`.
- Files committed here are intentional repo artifacts, not temporary placeholders.
