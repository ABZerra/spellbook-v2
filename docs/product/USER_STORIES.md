# User Stories (JTBD-Aligned)

## Story 1: Fast Queueing
As a player between sessions, I want to scan and queue candidate spells quickly, so I can capture ideas without entering a full planning workflow.

## Story 2: Intent-Driven Preparation
As a player preparing for session start, I want each queued spell to be marked as add, replace, or queue-only, so I can control exactly what applies.

## Story 3: Explicit Replacement Control
As a player, I want to choose the exact prepared spell to replace from the same list, so I avoid unintended swaps.

## Story 4: Stable Review Context
As a player, I want to open current prepared spells in an overlay while keeping queue visible behind it, so I can compare without losing context.

## Story 5: Reliable Sync Hand-off
As a player using the extension, I want the final applied changes to sync with explicit acknowledgement, so I can trust what was committed.
